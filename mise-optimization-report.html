SiteReason it remains
_dispatch → detected prefix:task The task name exists only after runtime marker detection and language fragments are optional. Profiles remain sequential while each child uses its native internal graph.
haskell:install → haskell:update Runs only when Cabal's index cache is absent; an unconditional declarative dependency would change behavior and add network work.
Root tester wildcard wrappers (all, check, isolated) mise 2026.7.5 executes monorepo path references but its validator rejects them in depends and structured run. The wrapper also retains proven env -u GOROOT -u GOTOOLDIR sanitation.
install, lock, doctor, tasks Intentional public wrappers around mise commands, not aggregate task orchestration.
shell:run, md:run, ts:run User-facing argument proxies remain available; fixed child tasks now call their tools directly.

Remaining limitations and deliberate non-changes

  • Monorepo task paths are kept behind one documented string wrapper until mise's validator resolves //… references in native task relationships. Execution, attribution, wildcard selection, and bounded scheduling were all verified on 2026.7.5.
  • jobs = 2 bounds each scheduler and the monorepo layer to two top-level fixture jobs. Optional-profile child schedulers mean total subprocess concurrency can be higher, but remains bounded.
  • mise install --monorepo was not adopted: eagerly installing every heavyweight ecosystem does not simplify normal fixture execution or improve its measured path.
  • No sources/outputs were added. The candidate gates are tests, linters, audits, secret scans, lock-currentness checks, or package-manager installs whose validity cannot be proven by a stable output timestamp.
  • No environment-specific gate was skipped; all host fixtures and the representative Dagger gate passed.
